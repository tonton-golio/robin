Parallel sub-agent sweep

For repetitive audits across a directory tree, dispatch one sub-agent per directory (or per directory bundle). Each sub-agent works in parallel with a clear, narrow scope.

When this applies

  • Auditing a large brain or codebase where each subdirectory has independent content.
  • Tasks where work is largely mechanical but volume is high (e.g., link audits, frontmatter conformance).
  • Tasks where the main-context agent benefits from staying lightweight.

How it works

  1. Identify the directories to cover. Bundle small ones together.
  2. Dispatch one sub-agent per bundle. Give each strict bounds:
    • Stay within the assigned directory.
    • Mechanical fixes only (no semantic restructures).
    • No deletes; no commits.
    • Flag semantic decisions to the orchestrator.
  3. Collect results in the main context. Triage flagged items.

Why it works

  • Parallelism keeps wall-clock time low.
  • Strict bounds prevent runaway sub-agents from making unsafe edits.
  • Main context stays free for triage.

Watch for

  • Sub-agents going outside their scope. Verify bounds in the dispatch prompt.
  • Over-reporting. Encourage terse output ("X fixed, Y flagged"), not full diffs in the response.
