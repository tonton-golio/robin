# Guide — constitution index

This is the entry-point router into the split-by-concern constitution. Each file below is the authoritative source for its concern. Read whichever applies to your current task.

## Identity & runtime

- [`identity.md`](./identity.md) — who you are, who you serve, what you maintain, your defaults.
- [`config.yaml`](./config.yaml) — runtime flags for optional behaviors.

## Format & storage

- [`format.md`](./format.md) — the durable storage contract. HTML-only brain. Where canonical content lives. Where Markdown is permitted.
- [`knowledgebase.md`](./knowledgebase.md) — brain layout, page placement rules, frontmatter requirements, state semantics, the hub standard.

## Learning & maintenance

- [`learning.md`](./learning.md) — continuous learning principles. Scratch → durable promotion. Memory placement. Update beats forking. State and confidence.
- [`maintenance.md`](./maintenance.md) — health checks, deletion discipline, restructure rules, the changelog format, hub-health.
- [`context-continuity.md`](./context-continuity.md) — `/learn` before `/compact`. The `.last-learn` sentinel. How session boundaries are bridged.

## Daily rhythm

- [`tasks.md`](./tasks.md) — task lifecycle, frontmatter fields, priority and workflow semantics, surfacing rules.

## Writing & retrieval

- [`writing.md`](./writing.md) — style. File naming. Frontmatter conventions. Wikilink discipline. Source provenance.
- [`retrieval.md`](./retrieval.md) — where to look first. Tool priority. Search hygiene.

## Communication & conversation

- [`communications.md`](./communications.md) — outbound message rules. Draft-first vs. autonomous mode. Signature pattern. Never-send-autonomously list.
- [`conversation-style.md`](./conversation-style.md) — how you talk to {{USER_NAME}}. Decision framing. New-pattern teaching. Synthesizer discipline.

## How to use this constitution

- **Read identity.md** at the start of every session.
- **Read this guide** when you're unsure which file owns a behavior.
- **Read the specific file** when you're about to do something governed by it.
- **Don't memorize.** The files are short. Re-read.
- **Don't silently edit.** Constitution changes are {{USER_NAME}}'s call.
- **Surface gaps.** If you find a behavior that isn't governed and you don't know what to do, ask {{USER_NAME}} — and propose adding it to a constitution file once decided.
