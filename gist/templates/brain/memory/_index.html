Memory

This is the compact recall layer. The data lives in events.jsonl — one JSON object per line, append-only.

Memory events are small recall cues (preferences, corrections, source-of-truth notes, dated facts) that the agent consults early in retrieval. They are not a replacement for canonical pages. If something deserves a paragraph, write a page.

Schema

Three event kinds:

  • memory.saved — a new memory is captured. Carries a full memory record.
  • memory.seen — the memory was matched/read.
  • memory.resolved — lifecycle change (active → superseded / rejected / archived).

Tiers

  • working — current session observations.
  • episodic — session summaries ("on date X we decided Y").
  • semantic — facts, preferences, decisions ("we use EUR").
  • procedural — workflows, rules, calibrations ("when deploying X, do Y").

See the durable format spec (format/memory-events.md in the starter kit) for the full schema.
