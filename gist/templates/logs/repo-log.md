# Repo log

Append-only record of repo/tooling changes — framework edits, dependency bumps, build/infra tweaks, and other operational changes to the code (as distinct from brain knowledge in `changelog.md`).

## Entry format

```markdown
## YYYY-MM-DDTHH:MM:SSZ — <area> — <one-line summary>
what: <what changed and why>
paths: <files/dirs touched>
```

**Areas:** `app` | `indexer` | `mcp` | `converter` | `scripts` | `hooks` | `deps` | `ci` | `config`.
