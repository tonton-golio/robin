# Robin

**A starter kit for building your own agentic second brain** — a durable, local-first knowledge base that an AI coding agent (Claude Code, Cursor, and friends) can read, write, and reason over, plus a daily working rhythm to keep it healthy.

![The Robin web UI — your daily dashboard over a local, file-based vault. (Synthetic demo data.)](docs/screenshot.png)

Robin is not a product. It's a **pattern**: a set of conventions, a file format, an MCP server, a local web UI, and a library of agent skills you copy into a repo to give your agent a real memory.

## What's in here

This repo is laid out the way you'll actually use it — **fork it and go.** The framework lives under `robin/`, the repo root holds the glue, and your data lives in a vault directory (`base/` by convention) that setup creates.

```
robin/             The framework (the only part you share / that upstream PRs touch):
├── app/           Next.js web UI, the Markdown→HTML converter, the file indexer
│                  (SQLite FTS + optional vectors), and the MCP server.
├── gist/          The starter kit: operating model, file-format spec, and
│                  templates for the constitution, skills, and brain scaffolding.
│                  → robin/gist/setup.md is written for your agent to execute.
├── scripts/       Operational helpers (doctor.sh — a vault health audit).
└── AGENTS.md      A note for AI agents working in the framework.

Makefile           `make robin-ui`, `make doctor` — run from the repo root.
.gitignore         Keeps secrets + runtime state out of git; your brain stays tracked.
docs/  LICENSE     Screenshot + MIT license.

base/              ← YOUR VAULT (created by setup; the name is set by ROBIN_VAULT).
CLAUDE.md  .claude/  .mcp.json   ← generated by robin/gist/setup.md from the kit's
                   templates: CLAUDE.md + .claude/ are your agent's control plane,
                   .mcp.json wires the MCP server to your vault.
```

Your data lives in the **vault** directory — separate from the `robin/` framework. That separation is what lets you share or upstream the framework without leaking anything personal.

## Quickstart

Robin has two flavors. Start **lightweight** (just files + your agent), then add the **app** when you want a browser UI and indexed search.

**Prerequisites:** Git and an AI coding agent (Claude Code or Cursor). For the optional app, also Node 24+ and npm (see [`robin/app/.node-version`](./robin/app/.node-version)).

### 1. Get the repo

**Fork** this repo (or use it as a template), then clone your fork:

```bash
git clone https://github.com/<you>/robin.git my-second-brain
cd my-second-brain
```

You already have the right layout: the framework at `robin/`, the glue (`Makefile`, `.gitignore`) at the root. Your personal data will live in a vault directory the next step creates — `base/` by convention.

### 2. Let your agent scaffold the vault + control plane

Open [`robin/gist/setup.md`](./robin/gist/setup.md) with your agent and tell it:

> **"Set up Robin using this guide."**

That file is written to be read and executed by an AI agent. It asks a few questions (your name, the agent's name, the vault directory, your time zone), then creates your `CLAUDE.md`, the `.claude/` control plane (constitution + skills + hooks), a seeded `base/brain/`, and — if you want the app — `.mcp.json`. **That is the entire lightweight flavor**: files + your agent, no server, no database.

### 3. (Optional) Stand up the app — UI, indexed search, MCP

For the browser UI, indexed search, and an MCP server your agent can call, from the repo root:

```bash
cp robin/app/apps/web/.env.example robin/app/apps/web/.env.local   # then edit it: set ROBIN_VAULT
cd robin/app && npm install                                        # install all workspaces
npm run build --workspace=@robin/mcp-server --workspace=@robin/indexer --workspace=@robin/converter
cd -                                                               # back to repo root
cp robin/gist/templates/mcp.json .mcp.json                         # then set {{REPO_ROOT}} + {{VAULT_DIR}}
make robin-ui                                                      # → http://localhost:8400
```

`.mcp.json` is what lets Claude Code call the Robin MCP server; after it exists, reconnect the MCP in your agent (`/mcp` in Claude Code). The full, copy-pasteable walkthrough — every env key and the exact wiring — is in [`robin/gist/app-setup.md`](./robin/gist/app-setup.md).

### 4. Verify the wiring

```bash
make doctor          # vault has brain/, secrets gitignored, page format, logs present, ...
```

`make doctor` works for **both** flavors: on the lightweight (file-only) setup it skips the MCP check cleanly; with the app it also validates `.mcp.json`.

> **The one variable that matters: `ROBIN_VAULT`.** It's the path to your vault (`base/`, created in step 2). The web app, the MCP server, and `doctor.sh` all read it. The shipped `Makefile` defaults it to `./base`; override per command with `make robin-ui ROBIN_VAULT=/abs/path`, or set it in `robin/app/apps/web/.env.local` and `.mcp.json`. Keep all three in agreement, or they'll point at different vaults.

## Contributing

Robin is built to be improved by the people — and the agents — who use it. If you hit a bug or have an idea, please send it back:

- **Open an issue or pull request** here. Keep changes scoped to the framework (`robin/`); never include personal or company data from your vault.
- **AI agents:** there's a note written for you at [`robin/AGENTS.md`](./robin/AGENTS.md) — when you spot something worth fixing, offer to open a PR, but always check with your human first.
